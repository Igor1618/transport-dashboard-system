# Умная логика объединения машин с похожими номерами

## Проблема

Старая логика объединяла машины **только по похожим номерам**, что приводило к ошибкам:

- `B083TM43` и `B083TH43` объединялись, хотя это **РАЗНЫЕ машины**
- Они работали **одновременно** с **разными водителями** на **разных маршрутах**
- Результат: неправильная статистика, завышенные показатели

## Решение

Новая логика использует **контекстную информацию** из данных Excel:
- Номер машины
- Водители
- Маршруты
- Даты работы

### Алгоритм

#### Шаг 1: Нормализация номеров
```javascript
normalizeVehicleNumber("В840ТО43") → "B840TO43" // Кириллица → Латиница
normalizeVehicleNumber("b840to43") → "B840TO43" // Нижний → Верхний регистр
```

#### Шаг 2: Поиск похожих номеров
Два номера считаются похожими если:
1. После нормализации они **идентичны** (B840TO43 = В840ТО43)
2. Расстояние Левенштейна = 1 при одинаковой длине (B083TM43 ≈ B083TH43)

#### Шаг 3: Проверка контекста

Для каждой пары похожих номеров проверяем:

##### ❌ НЕ объединять если:
- **Нет общих водителей** И **нет общих маршрутов**
  ```
  B083TM43: водители [Иванов], маршруты [Москва-СПб]
  B083TH43: водители [Петров], маршруты [Москва-Казань]
  → РАЗНЫЕ машины
  ```

- **Временной конфликт**: работали в один день с разными водителями
  ```
  2024-12-15:
    B083TM43 → Иванов
    B083TH43 → Петров
  → Обе машины работали одновременно, это РАЗНЫЕ машины!
  ```

##### ✅ Объединять если:
- **Есть общие водители** ИЛИ **есть общие маршруты**
- **Нет временных конфликтов**

```
В840ТО43: водители [Иванов, Сидоров], маршруты [Москва-СПб]
B840TO43: водители [Иванов], маршруты [Москва-СПб]
→ ОДНА машина (общий водитель, общий маршрут, нет конфликтов)
```

## Примеры работы

### Пример 1: Кириллица vs Латиница (объединяем)
```
Машина 1: В840ТО43 (кириллица)
  - Водитель: Иванов А.С.
  - Маршрут: Москва-СПб
  - Даты: 01.12, 05.12, 10.12

Машина 2: B840TO43 (латиница)
  - Водитель: Иванов А.С.
  - Маршрут: Москва-СПб
  - Даты: 15.12, 20.12, 25.12

Решение: ✅ ОБЪЕДИНИТЬ
Причина: Номера идентичны после нормализации, один водитель, один маршрут, даты не пересекаются
```

### Пример 2: Опечатка но разные машины (НЕ объединяем)
```
Машина 1: B083TM43
  - Водители: Иванов А.С.
  - Маршрут: Москва-СПб
  - Даты: 01.12, 05.12, 10.12

Машина 2: B083TH43
  - Водители: Петров В.И.
  - Маршрут: Москва-Казань
  - Даты: 01.12, 05.12, 10.12

Решение: ❌ НЕ ОБЪЕДИНЯТЬ
Причина: Разные водители, разные маршруты, работали одновременно (01.12, 05.12, 10.12)
```

### Пример 3: Один водитель работает на двух похожих машинах (объединяем осторожно)
```
Машина 1: B869TK43
  - Водитель: Иванов А.С.
  - Маршрут: Москва-СПб
  - Даты: 01.12-15.12

Машина 2: в869тк43 (кириллица, нижний регистр)
  - Водитель: Иванов А.С.
  - Маршрут: Москва-СПб
  - Даты: 16.12-31.12

Решение: ✅ ОБЪЕДИНИТЬ
Причина: Номера идентичны после нормализации, один водитель, один маршрут, даты не пересекаются
```

### Пример 4: Временной конфликт (НЕ объединяем)
```
Машина 1: B123CD45
  - Водитель: Иванов А.С.
  - Дата: 15.12.2024, 10:00

Машина 2: в123cd45 (кириллица)
  - Водитель: Петров В.И.
  - Дата: 15.12.2024, 14:00

Решение: ❌ НЕ ОБЪЕДИНЯТЬ
Причина: В один день (15.12) работали РАЗНЫЕ водители. Даже если номера похожи,
физически машина не может быть в двух местах одновременно с разными водителями.
```

## Логи работы алгоритма

При запросе `/api/vehicles/stats` в консоль выводятся логи:

```
🚗 Найдено 150 уникальных номеров в БД

❌ B083TM43 и B083TH43 - РАЗНЫЕ машины (2024-12-15: разные водители)
✅ В840ТО43 и B840TO43 - ОДНА машина (общие признаки, нет конфликтов)
🔗 Объединяем 2 номеров: В840ТО43, B840TO43

✅ Итого 149 машин после умного объединения
```

## Преимущества новой логики

1. **Точность**: Не объединяет разные машины с похожими номерами
2. **Контекст**: Использует реальные данные (водители, маршруты, даты)
3. **Безопасность**: Конфликты приводят к НЕ объединению (лучше не объединить, чем объединить неправильно)
4. **Прозрачность**: Логи показывают причины решений

## Внедрение

### Вариант 1: Заменить текущий файл
```bash
cp backend/routes/vehicles-improved.js backend/routes/vehicles.js
pm2 restart transport-backend
```

### Вариант 2: Использовать параллельно для тестирования
В `backend/server.js` добавить новый роут:
```javascript
const vehiclesImproved = require('./routes/vehicles-improved');
app.use('/api/vehicles-smart', vehiclesImproved);
```

Тестировать: `GET /api/vehicles-smart/stats?month=2024-12`

После успешного тестирования заменить основной роут.

## Тестирование

1. Запросить статистику: `GET /api/vehicles/stats?month=2024-12`
2. Проверить логи в консоли
3. Сравнить результаты со старой логикой
4. Убедиться что машины с разными водителями не объединяются

## Поддержка

При возникновении вопросов или неправильного поведения проверьте логи - они покажут почему машины были объединены или разделены.
