## О проекте

**Название:** Дашборд Транспортной Компании
**Описание:** Компактный веб-дашборд для управления транспортной компанией с микросервисной архитектурой

## Технологический стек

### Frontend
- **HTML5** - структура страницы
- **CSS3** - стилизация (Grid, Flexbox, CSS Variables)
- **JavaScript (ES6+)** - логика приложения
- **Chart.js** - визуализация графиков

### Backend / Database
- **Supabase** - PostgreSQL база данных (http://195.26.226.37:8000)
- **PHP 7.4+** - опциональные API endpoints (enhanced-coordinator-v2.php, export-csv.php)

### Архитектура
- Статический HTML с встроенными стилями и скриптами
- Микросервисная структура (отдельные HTML файлы для разных функций)
- Real-time данные через Supabase Client

## База данных (Supabase)

### Таблицы

#### `vehicles` (Транспортные средства)
- `id` - UUID, первичный ключ
- `number` - номер транспорта (например, "Н678МН78")
- `model` - модель транспорта (например, "MAN TGX")
- другие поля...

#### `vehicle_monthly_data` (Месячные показатели транспорта)
- `year` - год
- `month` - месяц (1-12)
- `income` - доход
- `expenses` - расходы
- `trips` - количество рейсов
- `efficiency` - эффективность (0-100%)
- связь с `vehicles` через внешний ключ

#### `drivers` (Водители)
- Таблица с информацией о водителях
- Используется для карточки "Всего водителей"

## Правила разработки

### 1. Стиль кода
- **Отступы:** 4 пробела
- **Кавычки:** одинарные `'` для JavaScript, двойные `"` для HTML атрибутов
- **Именование:**
  - CSS классы: kebab-case (`.kpi-card`)
  - JavaScript переменные: camelCase (`vehiclesData`)
  - Функции: camelCase (`updateDashboard()`)

### 2. CSS
- Использовать существующие цвета:
  - Градиент: `#667eea` → `#764ba2`
  - Успех: `#10b981` (зеленый)
  - Ошибка: `#ef4444` (красный)
  - Фон: `#f5f7fa`
  - Белый блок: `#ffffff`
  - Текст: `#1f2937`
  - Серый текст: `#6b7280`
  - Рамки: `#e5e7eb`

- Переиспользовать существующие классы:
  - `.block` - белый блок с тенью
  - `.block-title` - заголовок блока
  - `.loading` - индикатор загрузки
  - `.error` - сообщение об ошибке

### 3. JavaScript
- Асинхронные запросы через `async/await`
- Обработка ошибок через `try/catch`
- Функция форматирования: `formatNumber(num)` для отображения чисел с ₽
- Глобальные переменные:
  - `currentMonth` - текущий месяц (0-11)
  - `currentYear` - текущий год
  - `vehiclesData` - массив данных транспорта

### 4. Supabase запросы
```javascript
// Пример запроса с join
const { data, error } = await supabase
    .from('vehicles')
    .select(`
        *,
        vehicle_monthly_data (
            year,
            month,
            income,
            expenses
        )
    `);

if (error) throw error;
```

### 5. Добавление новых KPI карточек
1. Добавить HTML в `.kpi-row` (строка 398-423)
2. Добавить ID для значения (`id="kpiNewValue"`)
3. Создать/обновить функцию загрузки данных
4. Обновить значение: `document.getElementById('kpiNewValue').textContent = value`

### 6. Адаптивность
- Использовать CSS Grid для layout
- Поддерживать мобильные браузеры
- Минимальная ширина: 320px

### 7. Производительность
- Минимизировать запросы к БД
- Кэшировать данные в `vehiclesData`
- Обновлять только измененные элементы DOM
